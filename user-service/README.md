# User Service

A microservice for managing user data with both HTTP REST API and microservice capabilities.

## Features

- **Dual Mode Operation**: Supports both HTTP REST API and Redis microservice communication
- **User Management**: Get and update user information
- **Prisma Integration**: Type-safe database operations
- **Validation**: Input validation with class-validator
- **CORS Support**: Cross-origin resource sharing enabled

## API Endpoints

### HTTP REST API (Port 3002)

- `GET /api/v1/users/:id` - Get user by ID
- `PUT /api/v1/users/:id` - Update user information

### Microservice Messages (Redis)

- `{ cmd: 'get_user' }` - Get user by ID
- `{ cmd: 'update_user' }` - Update user information

## Data Model

```typescript
interface User {
  id: number;
  email: string;
  firstName?: string;
  lastName?: string;
  role: string;
  createdAt: Date;
}
```

## Usage

### Development
```bash
npm install
npx prisma generate
npm run start:dev
```

### Production
```bash
npm run build
npm run start:prod
```

## Integration with API Gateway

The user service integrates with the API Gateway through HTTP endpoints:
- Gateway calls: `http://user-service:3002/api/v1/users/*`
- Supports JWT authentication via Authorization header
- Returns structured JSON responses

## Database Schema

```prisma
model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  firstName String?
  lastName  String?
  role      String   @default("USER")
  createdAt DateTime @default(now())
}
```
